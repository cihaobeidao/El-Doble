### 2026-04-27 · 新增《事件范式》工艺规范文档 · 确认
- **决定**:在仓库根新增 `09_事件范式.md`，锁住单个事件页的标准格式——头部 7 必填字段（事件 ID / 类型 / Acto / 触发条件 / 触发时刻 / 时辰消耗 / Tesoro 消耗）+ 8 个可选字段（死线 / 反应窗口 / 前置/后续/互斥事件 / 重玩门槛 / 版本 / 主笔）+ 正文三大区块（设计目标 / 场景 / 选项表 / 写作笔记）+ 自检清单。范围:**写作向**（不深入 JSON schema），但头部字段名按"将来 1:1 映射到 `demo/content/events/*.json`"的思路命名。
- **理由**:03 §四 与 07 三个样本已经收敛到近乎一致的格式但有微差——早期把页面格式锁住，后期写几十上百个事件不返工 + AI 写草稿不漂移 + 引擎落地时只需 markdown 头部转 JSON。
- **影响**:
  - 工艺层而非设定源——`09_事件范式.md` §10 明确写了"不规定世界 / 剧情 / 角色，只规定页面格式"。
  - 与设定源冲突时**以源文档为准**（§8、§10）。
  - 新增字段或改字段顺序属 CLAUDE.md §3 "先问后做"。
  - **CLAUDE.md §4 文档地图待补一行**——Keyan 自行决定何时加（Claude 不擅自改 CLAUDE.md）。
- **维护**:文档内不保留 changelog / 版本历史；早期阶段（M2 之前）改具体事件向范式看齐，后期再评估反向调整成本。
