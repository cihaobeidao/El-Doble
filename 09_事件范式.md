# 09 · 事件范式

> 这份文件锁住**单个事件页长什么样**——头部元数据、正文模块顺序、选项表格、写作笔记、自检清单。
> 与其他文档的分工：
> - `03_核心玩法.md` §四 是事件的**概念定义**（六类、时辰消耗、可见性原则）
> - `04_数值与阵营.md` 是数值轴与派系态度的**语义**
> - `06_主题与调性.md` §二 是写作的**硬规则**
> - `07_风格样本.md` 是已成稿的**金标准**
> - 本文是把以上四者落到"一张事件页应当怎样排版"的**形态规范**
>
> 本范式**只规定排版**，不规定内容。任何与设定源冲突的事项，**以源文档为准**。

---

## 一、为什么要范式

之前几份样本（03 §四 的 ACTO_III_C_002，07 的 PROL_C_001 / ACTO_II_P_004 / ACTO_IV_P_007）已经收敛到一套近乎一致的页面布局，但**字段顺序、可选项的标注方式、选项表格列名**仍有微差。后期会有几十甚至上百个事件——**早一点把页面格式锁住**，能让：

1. **翻阅时不动脑**——看到任何事件页，眼睛知道下一段是什么。
2. **AI 写作不漂移**——给 AI 一个事件 ID 与几行设计意图，它能直接套出格式正确的草稿。
3. **后期落引擎不返工**——头部字段名与"事件类型"枚举值后期会 1:1 映射到 `demo/content/events/*.json` 的键。**先把字段名定下来，将来引擎层只是把 markdown 头部转成 JSON**，不用全文重写。

---

## 二、事件页的三大区块

按**固定顺序**呈现，不要打乱：

```
┌─────────────────────────────────────────┐
│ 1. 头部元数据块（YAML 风格七行 + 可选项）│
├─────────────────────────────────────────┤
│ 2. 正文区块                              │
│    2.1【设计目标】（可选但强烈推荐）     │
│    2.2【场景】（可分多个子场景）          │
│    2.3【触发：XXX】（限时危机用）         │
│    2.4【选项】（统一表格）                │
│    2.5【对话余波】/【尾声】（可选）        │
├─────────────────────────────────────────┤
│ 3. 写作笔记 / 团队对齐要点（事件末尾）    │
└─────────────────────────────────────────┘
```

下面逐块锁住。

## 三、头部元数据块

### 3.1 必填项（7 行，永远按此顺序）

```
**事件 ID**: <ID>
**事件类型**: <Audiencia / Público / Decreto / Crisis / Íntimo / Archivo>
**所属 Acto**: <序幕 / 第一幕 ... / 第六幕 / 终章>
**触发条件**: <state / flag / 派系态度的复合表达，自然语言>
**触发时刻**: <第 N 天 HH:MM 或"任意空白时段"或"游戏开始">
**时辰消耗**: <数字 或 范围 或"0（XX 特例）">
**Tesoro 消耗**: <数字 或 0>
```

### 3.2 可选项（按需追加在必填项之后，每行一项）

```
**死线**: <T 时之前 / N 时辰内>            ← Crisis 类常用
**反应窗口**: <X 秒>                        ← 限时即时事件用（参 07 样本 3）
**前置事件**: <ID 或 ID 列表>               ← 必须先发生过
**后续事件**: <ID 或 ID 列表>               ← 本事件触发链入队
**互斥事件**: <ID 或 ID 列表>               ← 本事件发生则锁死
**重玩门槛**: <Verosimilitud / Cordura / 标记 阈值> ← 解锁档案选项的条件
**版本**: <草稿 / 已审 / 锁定>              ← 写作流程态
**主笔**: <人名>                            ← 仅草稿期标注
```

### 3.3 事件 ID 命名规则（强制）

格式：`<ACTO 标识>_<类型简称>_<三位序号>`

- **ACTO 标识**：`PROL`（序幕） / `ACTO_I` / `ACTO_II` / `ACTO_III` / `ACTO_IV` / `ACTO_V` / `ACTO_VI` / `EPILOG`（终章）
- **类型简称**（与 03 §四 一致）：`A` Audiencia · `P` Público · `D` Decreto · `C` Crisis · `I` Íntimo · `AR` Archivo
- **序号**：三位数从 001 起，**按 Acto 内的写作顺序递增**，不留空号

**例**：`PROL_C_001` · `ACTO_II_P_004` · `ACTO_IV_P_007` · `EPILOG_I_002`

### 3.4 字段填写约定

- **触发条件**用自然语言写，不要 if/else 伪代码。例："Orden ≤ −20 且 Calle ≥ 警惕"。
- **时辰消耗**支持范围（"1–3"）与"0（XX 特例）"。**永远不要写"自动"或"未定"**——拿不准就写"0（待定，写作期占位）"并加 `// TODO`。
- **Tesoro 消耗**默认 0；只有触发"主动政策"类事件才扣。
- **派系 / 角色名一律保留西语原文**——`Doña Isabel` 不是"伊莎贝尔夫人"，`十二姓氏` 与 `Los Doce` 在叙事文本里都可以，但**头部字段统一西语**。

## 四、正文区块

### 4.1 【设计目标】（强烈推荐，★★★★ 事件以上**必填**）

紧跟在头部元数据之后，单独一段，用引用块开头：

```
> **设计目标**：在 8 分钟内让玩家感受到——
> 1. Saulito 是个具体的人，不是符号
> 2. Vega 的逻辑是冷的，不是反派的冷
> 3. 黑色幽默从最微小的细节里冒出来
> 4. 玩家的"选择"是假的——这本身就是本作的命题
```

- **1–4 条**，每条一句话。
- 回答"为什么这个事件存在 / 它在 Acto 弧上做什么 / 玩家应当带走什么感受"。
- **不写功能描述**（"让玩家熟悉时辰系统"是反例）；写**情感锚点**与**主题命中**。

### 4.2 【场景】

主体叙事段落。**严格遵守 06_主题与调性 §二 原则 1：三种声音不混在同一段**。

#### 排版规则

| 声音 | 排版 | 示例 |
|---|---|---|
| **场景叙述者**（第三人称） | 常规段落 | `凌晨四点十七分。Las Veladas 街区。` |
| **Saulito 内心独白**（第二人称） | `> *引用块 + 斜体*` | `> *你右脚的鞋底松了。*` |
| **新闻 / 报纸 / 广播** | `> 引用块（不加斜体）` | `> Radio Aurora，七月十五日午间播报：……` |
| **NPC 对白** | `— "破折号引语"` | `— "Patrón，您先看这个。"` |
| **其他替身的内心独白**（多周目） | 各自人称，参 `角色档案/00_主角线/<替身>.md` 声音卡 | Catalina 用第一人称广播稿；Tobías 用第三人称传记体 |

#### 子场景拆分

一个事件可以含多个子场景，用 `**【场景】**` / `**【车内】**` / `**【尾声】**` 等粗体方括号分段。**子场景标题用名词或地点，不用功能描述**（"【NPC 离场】"是反例，"【车停了】"是好的）。

#### 反例（看到就改）

- ❌ 在第三人称段落里突然冒一句斜体内心独白（要拆段）
- ❌ 用形容词代替物件（"夜色凝重" → 改成具体的物件 / 声音 / 气味）
- ❌ 旁白评论自己的幽默（"哈哈，多么讽刺啊"——**永远不写**）
- ❌ 旁白点破隐喻（让玩家自己发现）
- ❌ Saulito 内心独白冒出"民粹主义""寻租"这种词（除非他在复述别人）

### 4.3 【触发：XXX】（仅 Crisis / 限时事件用）

倒计时事件用一个独立段落标记触发瞬间。例：

```
**【触发：Mariposa】**

仪仗队走到你的车前八米时——

人群里冲出一只狗。

[…]

—— **30 秒倒计时开始** ——
```

- 触发瞬间必须**单独成段**，与之前的铺垫分开。
- 倒计时用 `—— **X 秒倒计时开始** ——` 的格式（双破折号 + 粗体）。

### 4.4 【选项】

#### 统一表格格式

```markdown
**【选项】**

| 选项 | 说什么 / 做什么 | 数值/标记效果 |
|---|---|---|
| **A** | "把钱拿回去。我自己上车的。" | *Cordura* +5；隐性标记 `WALKED_IN_DIGNIFIED` |
| **B** | （沉默几秒）"……我女儿下周末过生日。" | *Cordura* −5；隐性标记 `MENTIONED_LUCIA_TO_VEGA`（第六幕变量） |
| **C** | "好。但合同里加一条……" | *Cordura* 不变；Vega 青年派起始信任 +5；隐性标记 `NEGOTIATED_EXIT_CLAUSE` |
```

#### 列约定

- **第 1 列 选项**：`**A**` / `**B**` / `**C**` / `**D**`，最多 4 个常规选项 + 1 个档案选项。
- **第 2 列 说什么 / 做什么**：玩家**实际看到**的选项文案。短句、名词为主、避免功能性命名。可以是直接引语 `"……"`，也可以是动作 `（沉默几秒）"……"` 或纯描述 `示意保安抱走她`。
- **第 3 列 数值/标记效果**：所有显性 + 隐性后果。**这一列玩家看不到**——它是给写作 / 审稿 / 引擎用的。

#### 数值/标记效果的标注规范

| 类别 | 写法 | 示例 |
|---|---|---|
| **Tier 1 国家指标** | `Economía / Orden / Legitimidad / Tesoro ±N` | `Orden +15` |
| **Tier 2 派系态度** | `<派系名> ±N` 或 `<派系名> <文字标签>` | `十二姓氏 +10` 或 `Doce → 试探合作` |
| **Tier 3 个人状态** | 用斜体 `*<指标>*` 区别于 Tier 1 / 2 | `*Verosimilitud* +5` `*Cordura* −20` |
| **Sospecha**（派系怀疑度） | `<派系名> *Sospecha* ±N` | `Mariana *Sospecha* +5` |
| **隐性标记** | 全大写 + 下划线 + 反引号 | `` `MENTIONED_LUCIA_TO_VEGA` `` |
| **解锁后续** | `解锁后续: <事件 ID>` | `解锁后续: ACTO_III_A_005` |
| **互斥锁** | `锁死: <事件 ID>` | `锁死: ACTO_III_A_006` |
| **特殊条件** | `检定 ≥ N 才能选 — 否则灰显` | `*Verosimilitud* ≥ 65 才能选 — 否则灰显` |

**多重效果之间用分号 `；` 分隔**。一行包含的独立后果建议**不超过 3 个**——超过就分行（用 `<br>` 在表格内换行，或拆成多行注解）。

#### 档案选项（多周目灰显选项）

格式：

```markdown
| **D** | （档案选项，仅当上一周目曾解锁 Mariana 北方履历后可见）"您写的硕士论文我读过……" | **第一周目灰显不可选**；隐性标记 `MARIANA_KNOWS_YOU_KNOW`；Mariana 当晚结束时会留下一张纸条 |
```

- 选项文案前用 `（档案选项，仅当 ___ 时可见）` 开头。
- 数值/标记列首条写 `**第一周目灰显不可选**`（如适用）。
- 解锁条件**写在头部元数据 §3.2 的"重玩门槛"字段里**，避免选项文案剧透。

#### "假选择"事件（无真选项）

某些事件（如 PROL_C_001）三个选项最终通向同一结局，差别在 *Cordura* 起始值与隐性标记。**必须在选项表上方加一段引用块说明**：

```
> 虽然玩家会得到三个选项按钮，**实际上每一个都通向同一个序幕结局**——差别在于起始 *Cordura* 与隐性标记。
> 这是本作"假选择"机制的第一次教学：**你不是在决定要不要做，你是在决定你以什么姿态接受**。
```

### 4.5 【对话余波】 /【尾声】（可选）

选项之后的额外场景或 cutscene，**无选项**。用于：

- 选项的即时反应（NPC 的下一句话、Saulito 的内心反应）
- 进入下一个事件 / cutscene 的转场
- 关键的"延迟反馈"信号（参 07 样本 2 末尾"她的眼神跟笑没有同步"）

排版同 §4.2 场景。结尾用 `[场景结束]` 或 `[过场：进入 XXX]` 标记出口。

## 五、写作笔记 / 团队对齐要点

事件页**末尾必带**这一节：

```markdown
**【写作笔记 / 团队对齐要点】**

1. <调性根 / 物件信号 / 节奏关键 / 常见误写——任选一类>
2. ……
3. ……
（3–5 条）
```

#### 内容指南

每条回答以下其中一个问题：

- **调性根**：哪个细节是这个事件的"灵魂"？删掉就不再是这个事件？
- **物件信号**：哪些物件在做政治信号工作？（手套布 vs 丝、表 1968 vs 1972、Yves Saint Laurent 香水……）
- **节奏关键**：某段重复 / 留白 / 倒计时 / 沉默是节奏装置，不是错误。
- **延迟反馈**：哪个隐性逻辑是事后才让玩家意识到的？（参 07 样本 2"我会让秘书安排"那段）
- **常见误写**：AI 或新主笔最容易把哪句改坏？

**写作笔记不出现在玩家可见 UI 里**——它是给协作者看的"作者意图说明"。引擎落地时**不消费这一段**，可以直接丢弃。

## 六、自检清单（写完一个事件后逐条勾）

写完一个事件，把下面清单复制到 PR 描述 / commit message / review 评论：

```
─────────────────────────────────────────
 事件 ID: ___________
 主笔: ___________ · 完成日: ___________
─────────────────────────────────────────
 调性自检（06_主题与调性 §二）
 □ 三种声音没有混在同一段
 □ 没有"贴脸笑"（旁白未评论自己的幽默）
 □ 没有旁白点破隐喻
 □ 没用"夜色凝重"等情绪化形容词替代物件
 □ Saulito 内心独白没出现"民粹主义""寻租"等政治术语
 □ NPC 对白能区分阶级（句长 / 词汇 / 信息密度）
 □ 派系 / 角色名保留西语原文，未中文化

 派系平衡自检（06 §四 + §六 红线 4）
 □ 涉及的派系没有被写成单纯"对的"或"错的"一边
 □ 出场的主要 NPC 至少命中"想认同 / 想讨厌 / 看错了"瞬间中的某一个
 □ 黑色幽默来自情境，不来自对受害者的嘲笑

 数值自检（04_数值与阵营）
 □ 没有"全好"或"全坏"的选项（任何选项至少一项明显代价）
 □ Verosimilitud 不单独决定一切
 □ 玩家可见 UI 不显示精确数字（除 Tesoro 之外，其他都是文字标签）
 □ 隐性标记拼写遵循全大写 + 下划线 + 反引号

 角色自检（角色档案/）
 □ 出场角色对白前已翻过对应"声音卡"与"写作红线"
 □ 没泄露 99_秘密文档.md 或 角色档案/99_秘密角色/ 的内容

 数据自检（CLAUDE.md §6 + §8）
 □ 文本不打算硬编码进 HTML / JS（事件文本属数据层）
 □ 头部 7 必填字段齐全且顺序正确
 □ 事件 ID 符合 §3.3 命名规则
─────────────────────────────────────────
```

## 七、最小完整范例（演示格式，非剧情骨架）

下面是一个**虚构**的 ★★★ 事件页，**仅作格式演示**，不属于任何 Acto 的实际剧情。

---

**事件 ID**: ACTO_I_A_003
**事件类型**: Audiencia
**所属 Acto**: 第一幕 *El Espejo*
**触发条件**: 第 3 天，玩家曾在第 2 天接见 Estela（标记 `MET_ESTELA_DAY_2`）
**触发时刻**: 第 3 天 11:00（默认时刻表项）
**时辰消耗**: 2
**Tesoro 消耗**: 0
**前置事件**: ACTO_I_A_001
**后续事件**: ACTO_I_D_002（若选 B）

> **设计目标**：让玩家第一次发现，"早班简报"的措辞不是中性的——Estela 选择放进去的事，与她省略掉的事，本身就是一种政治。

---

**【场景】**

雨。早班简报第三页是 Estela 的字迹——她写"上午 11 点，*Padre* Solano 来访"。第四页是空白。

你昨天在档案里读过 Solano。Cardenal Soto 派下来的本堂神父，**不重要**，但他每周三上午来一次，带一份手写的"教区民情周报"。

> *档案上写真总统每周三都见他。每次都见他。三年。*

> *你不知道为什么。*

Estela 在门口探头：

— "Patrón，*Padre* 已经在二楼候着了。要请他上来吗？"

> *她叫"Patrón"叫得很自然。她叫真总统也叫"Patrón"吗？还是只叫你？*

— "请他上来。"

她点头出去了。**门没关严**。

---

**【选项】**

| 选项 | 说什么 / 做什么 | 数值/标记效果 |
|---|---|---|
| **A** | 让 *Padre* 把周报放在桌上，**不读**，谢谢他，请他下次再来 | *Verosimilitud* −5（真总统会读）；街头与教会 −3；隐性标记 `IGNORED_SOLANO_DAY_3` |
| **B** | 当面读完，问他"Cumbre Mayor 这周有几起家暴报案" | *Cordura* −2（你不该问得这么具体）；街头与教会 +5；解锁后续: ACTO_I_D_002 |
| **C** | 寒暄两句，把周报转交给 Vega"由军方酌情处理" | Junta-青年派 +3；街头与教会 −8；隐性标记 `OUTSOURCED_SOLANO` |

---

**【尾声】**

*Padre* Solano 走的时候，回头说了一句：

— "Patrón，您今天的眼神跟上周不一样。"

> *他知道吗？*
>
> *他不知道。他只是在客气。*
>
> *他不知道。*

[场景结束]

---

**【写作笔记 / 团队对齐要点】**

1. **门没关严**——这是 Estela 的小动作，不是疏忽。她想听你怎么对 *Padre* 说话。**这一句不能删**。
2. *Padre* 那句"您今天的眼神跟上周不一样"是延迟反馈装置——玩家此刻**应该**怀疑他识破了，但实际上他没有。这种"假识破"可以零星使用（全作不超过 4 次），用多了会麻木。
3. 选项 B 的"Cumbre Mayor 几起家暴报案"是**真总统不会问的问题**——这是 Saulito 街头本能的体现。*Verosimilitud* 没扣，但 *Cordura* 扣了——他**问出口的瞬间自己也意识到**了。
4. 不要把 Solano 写得贪婪或虚伪。他是 06 §二 原则 4 里"真心想认同他的瞬间"那种 NPC——他真的在送一份手写周报，三年没断。

## 九、何时不用本范式

本范式管的是**事件页**——单个有触发条件、占时辰、有选项（或假选项）的离散戏剧单元。下列情况**不走本范式**：

- **章节序场（cutscene）**：每个 Acto 开头的过场叙事——属于 `08_剧情/<Acto>.md` 的散文段落，不需要事件 ID 与选项表。
- **晚间简报模板**：*El Parte Vespertino* 是固定 UI 模板（参 03_核心玩法 §二），不是事件——它的内容由当日事件结果**自动组装**。
- **数值变化的"被动事件"**：例如 Economía 跌破 −60 触发的"恶性通胀总崩溃"事件链——它的**入口事件**是事件页，但事件链内的连续天数过场属于 cutscene。
- **多周目档案 / 卷宗解锁条目**：见 05_多周目机制——它们用自己的卡片格式，不混入事件页。
- **教程界面文本**：*Tarjetas Esenciales* 与 *El Expediente Báez* 走档案 UI 模板，不走事件页。

## 十、维护纪律

- **本范式属于工艺规范层**，与 `01_*.md`–`07_*.md`、`08_剧情/*`、`99_*.md` 平级，但**不是设定源**——它不规定世界、不规定剧情、不规定角色，只规定"页面长什么样"。
- 与本范式相关的改动（新增字段、改字段顺序、改自检清单）走 CLAUDE.md §3 "先问后做"流程。
- 文档内**不保留 changelog / 版本历史**——只保留当下生效的格式，历史用 `git log` 追溯。
- 范式与具体事件不一致时：
  - **早期阶段**（M2 之前）—— 改具体事件，向范式看齐。
  - **后期阶段**（事件量已大）—— 评估改范式还是改事件的成本，必要时与 Keyan 对齐。
